Tutorial

You are already familiar with the purpose of Albiruni 1.0, you have installed it, and started it.  Now do this:

Select Map Server

In the Map Servers panel, check a box under Select.   As you learn about the different data and colors on maps, or as server availability changes, select accordingly.

Opening Data

  • Decide where your interesting files are located.  
  • Albiruni can read GPS data from .gpx and .jpg files in a folder and everywhere under that folder (that is, an entire directory tree).  
  • For your first try, please test on a small collection, less than 1000 files or 1 million points.

Map of Installed Sample Data



Here is the gist of this tutorial.   When viewing the map,
  • Colored cells contain points in the data.
  • With the left mouse button, click/drag the map to pan it.
  • On the right are controls of appearance.  Hover over them to see a few words of explanation.
  • Fit zooms and pans the map to the smallest area covering all data.
  • Slide Zoom up to view the Earth closer.
  • Slide Precision up for smaller cells.
  • If you like your Precision, check Sync, and Precision/Zoom will change together for a consistent cell size as the map is zoomed.   Sync can restrict how far Precision or Zoom may change.

Hovering on Cells

  • Station the mouse pointer over any colored cell.  
  • It is highlighted with a yellow border, and source files are listed in the Details panel.  

Abiruni's mission is to help users answer questions with the map and these data details unlike any other software.

  • Details shows the breakdown of data from that file located in the cell. 
  • Double-click the file name (or row) to open it externally.  (Discussed below). 
  • If the yellow cell refers to only one file, you can also double-click the map in that cell.

Albiruni 1.0 displays three categories of GPS point counts:

  • Waypoints are assorted locations, usually deliberate and not automatic, possibly visited with a GPS, often descriptive, possibly marked on a map in standalone software.
  • Track points were visited with a GPS.    It also include points that Albiruni treats like an actual visit, such as a photo or other logged event that Albiruni may be able to read in the future.
  • Routes are normally created in software before visiting places.  (Routes are not expected to contain date/time.)

In this version, date and time are always coordinated universal time (UTC).   This is a helpful simplification because the UTC axis does not vary with location or date.   UTC has no hiccups. (See Appendix.)

Opening Data Externally

If you have applications installed and configured to read files such as .gpx and .jpg, double-click a row within Details to open the source file externally. 

  • For .gpx data, you will see the full extent of the original data, in whatever map is built into that application.  For example, Garmin BaseCamp can open .gpx.  
  • External applications do not know about Albiruni cells.  The user needs to navigate on an external map.  
  • Images such as .jpg can also open.  You may want a fast default viewer such as IrfanView.

Options

Albiruni 1.0 has these minimal options:
  • Data / Best Precision:   The precision of data maintained in memory and files, as approximate cell sizes in kilometers (km) or meters (m) between latitudes 45° north and 45° south.  The default is a maximum, 1.7 to 2.4 km (1 to 1.5 miles), geared toward efficiency.   If your data is spread over a relatively small area, you need to view smaller cells to answer questions, and your computer has enough memory and CPU speed, then choose a smaller Best Precision.
  • Map / Cell Color

The .albiruni folder

Albiruni creates a new folder .albiruni within the selected/opened folder.  There it stores summary data gathered from the input files.   It is a cache designed for speedup, avoiding parsing the original files often.  If you revisit the selected folder in the future, after input files were added or changed, more files under .albiruni will be added/changed.   In the future you can carefully delete it, in the interest of security, without affecting the original personal files.

OpenStreetMap

OpenStreetMap data is free and several servers are currently public/open/reliable.  But do not over-use this fine resource.  Your organization should run its own OpenStreetMap server for its own unlimited and private usage.   Review the links in the Map Servers panel.

Troubleshooting

If all of the following happen:

  1. an error or failure occurs,
  2. you cannot solve the problem,
  3. you are doing important work (or you are recruited to test Albiruni),
  4. and you want to report the problem, 
Then please use Help / Troubleshoot. 

Albiruni will be improved, considering the time available and the urgency of the issues.

Panels

Drag the title bar of a panel to float it independently.  Optionally, move the mouse to a panel icon and drop it:  For example, you could dock the Details panel at the top.  

If the appearance is spoiled, use the menu choice:  Window / Restore Default Placement.
 

Appendix

Cells are made by a mathematical division of Earth's surface.   The cells are not states, counties, townships, cantons, departments, oblasts, or prefectures.  They originate at latitude=0 and longitude=0, and they depend on Precision; otherwise they are not adjustable.  Geographers divided Earth into two hemispheres by latitude (at the equator -- northern and southern) and two arbitrary hemispheres by longitude (at the Prime Meridian -- eastern and western).  By the same token, Albiruni divides hemispheres by two again and again several times, resulting in quadrilaterals that span D° latitude and D° longitude.  The quadrilaterals containing data points are shown as the colored cells.   In fact, for integer k,
D = 360 * 2-k
For example, k=18.   Increasing Precision increases k, and makes D smaller.

Time:  Rather than UTC, other imaginable times could be

  1. Original device's local time, but it is unreasonable to summarize it by First and Last if there are hiccups i.e. cells spanning time zones and DST on or off.   A specific UTC offset exists for a given location and time, not a swath of space and time.   Even at a location, it's doubtful to compute.
  2. The Albiruni user's local time, but its UTC offset may differ from the mapped points/times, again due to time zones and DST.

Waypoints that were created in desktop software, not connected to any GPS, may still affect the First and Last times if the software inserted its "computer time".  This is unreliable and unwanted, but it may depend on the software.  (In the future, Albiruni may be able to clean up such data.)
